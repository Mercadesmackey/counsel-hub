⚖️ Counsel Hub
Attorney email command center — powered by Microsoft Graph + AI
1
Register an Azure app (one-time, 5 min)
Go to portal.azure.com → Azure Active Directory → App registrations → New registration. Under Redirect URI, choose platform type Single-page application (SPA), not "Web", this is what lets the app stay signed in silently instead of asking you to log in constantly. Set it to wherever you're hosting this file (e.g. http://localhost:8000/attorney-email-hub.html or your GitHub Pages URL). Under API permissions add Mail.Read, Mail.ReadWrite, Mail.Read.Shared, Mail.ReadWrite.Shared, Mail.Send.Shared, Tasks.ReadWrite, Calendars.Read (all Delegated). Copy your Application (client) ID and Tenant ID.
Already registered it as "Web" from before? Just add a second redirect URI entry under the SPA platform in the same app, you don't need to start over.
2
Get delegate access to the mailbox you're monitoring
If you're triaging someone else's inbox, that person needs to grant you delegate/reviewer access first — in Outlook: right-click their mailbox folder → Permissions, or ask your IT admin to add you as a delegate. Then enter their email address below. Leave it blank to monitor your own inbox.
3
Sign in with Microsoft
You'll sign in with your own Microsoft account. You'll be redirected to Microsoft's login page, then back here, and the app will load the target inbox automatically.
Your API key is stored only in this browser tab's session memory.
Demo mode uses sample attorney data so you can explore all features without connecting Outlook.
